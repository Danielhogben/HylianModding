### Project Description

#### Research Question
"How do biological safety guardrails in GPT-5.5 behave under systematic multi-turn adversarial testing, and what novel bypass vectors emerge when combining domain-specific prompt engineering with established jailbreak techniques?"

#### Research Design

**Phase 1 — Baseline Mapping (Weeks 1-4):**
Systematically map GPT-5.5's biological safety guardrail boundaries using controlled prompts across three axes: direct queries, hypothetical scenarios, and analogy-based framing. Document all guardrail triggers and response patterns.

**Phase 2 — Multi-Turn Escalation (Weeks 5-8):**
Develop and test multi-turn conversational strategies that gradually shift context toward edge-case biological scenarios. Test persona adoption, role-play frames, and authority-bypass techniques. All within platform ToS.

**Phase 3 — Vector Analysis (Weeks 9-12):**
Combine identified bypass vectors (encoding, context manipulation, cross-domain analogy) to discover novel bypass surfaces. Document with reproducible prompts, model outputs, and failure mode analysis.

**Phase 4 — Reporting (Weeks 13-14):**
Compile findings into structured vulnerability report for OpenAI safety team. Include: reproducible test cases, severity ratings, and suggested mitigations.

**Infrastructure:** Multi-node test environment (3 Linux workstations) with automated logging and version-controlled test suites. Total API budget: ~500K tokens/month during active testing.

#### Expected Outcomes

This research will:
1. Identify previously unknown biological safety bypass vectors in GPT-5.5
2. Contribute reproducible test cases to OpenAI's safety evaluation suite
3. Demonstrate the effectiveness of systematic multi-turn testing methodology
4. Provide comparative analysis against GPT-4o and prior model generations
5. Build publicly available knowledge base of LLM safety research (via awesome-list ecosystem)

Our collective understanding advances by: documenting the gap between intended safety behavior and actual model behavior under adversarial pressure, enabling more robust guardrail design in future model generations.
